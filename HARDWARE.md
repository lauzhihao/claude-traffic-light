# Claude 红绿灯 — 硬件采购 & 接线指南

## 一、采购清单（淘宝）

> 总价预估 **¥60–110**（含 3D 打印外壳）。下单前把每项的"关键词"复制进淘宝搜索即可。

### 1. 主控板 — Raspberry Pi Pico（已焊排针）⭐ 必买

- **淘宝搜索关键词**：`树莓派 Pico 已焊排针` 或 `Raspberry Pi Pico H`
- **认准型号**：`Pico H`（"H" = Headers 已焊接）或 `Pico 2 H`（新款，更快但价格略贵，本项目两款都够用）
- **价格**：¥25–40
- **数量**：1
- **避坑**：
  - 不要买 `Pico W`（带 WiFi 那款），本项目用不上 WiFi，多花钱
  - 不要买没焊排针的版本（标注 `Pico` 不带 H），否则你得自己焊 40 个引脚
  - 国产 RP2040 兼容板（如 `WeAct RP2040`）也行，但建议第一次就买原版省心

### 2. WS2812 灯珠 ⭐ 必买

**首选方案 — WS2812 单灯模块（带 PCB 板，已焊好杜邦线版本）**
- **淘宝搜索关键词**：`WS2812B 单灯模块 已焊线` 或 `WS2812 RGB 模块 5050 杜邦线`
- 一颗带 6 根线（IN 端 3 根：VCC/GND/DIN，OUT 端 3 根：VCC/GND/DOUT），可串联
- **价格**：¥3–8/颗
- **数量**：3 颗

**备选方案 — WS2812 灯串（独立小板，灯之间已用线串好）**
- **淘宝搜索关键词**：`WS2812 单灯灯串 5V` 或 `WS2812B 像素灯串`
- 一串 10–50 颗任选，剪下前 3 颗就行
- **价格**：¥10–20/串
- **优点**：完全不用焊接，每颗灯之间已经串好
- **缺点**：灯之间距离固定（一般 5–10cm），可能需要剪短重接

> 💡 **强烈建议优先选"已焊线"版本**，否则你需要再花 ¥30 买个烙铁套装。

### 3. 杜邦线 ⭐ 必买

- **淘宝搜索关键词**：`杜邦线 母对母 20cm 彩色`
- **价格**：¥5–10（一包 40 根，用不完留着以后玩）
- **数量**：1 包
- **用途**：把 WS2812 的 VCC/GND/DIN 插到 Pico 的引脚上

### 4. USB 数据线 ⭐ 必买

- **认准接口**：Pico 是 **Micro-USB**（不是 Type-C！容易买错）
  - Pico 2 也是 Micro-USB
  - 部分国产改版可能是 Type-C，看清楚商品页
- **必须是数据线，不能是只能充电的线**
- **淘宝搜索关键词**：`Micro USB 数据线 1m`
- **价格**：¥5–10
- 家里有安卓老充电线翻出来试试，能识别就不用买

### 5. 3D 打印迷你红绿灯外壳 ⭐ 必买

- **淘宝搜索关键词**：`迷你红绿灯模型 3D 打印` 或 `红绿灯摆件 定制`
- **两条路**：
  - **A. 买现成卖家的成品模型**（最省事）：直接搜"红绿灯模型"，挑一个 8–15cm 高的，内部空腔够大，¥30–60
  - **B. 自己提供 STL 文件代打印**：淘宝搜"3D 打印代打印"，把 STL 发过去，¥20–40
- **关键尺寸要求**（采购前一定确认）：
  - 内部空腔高度 ≥ 8cm（要塞下 Pico ~5cm + 留余量）
  - 三个灯位的孔径 ≥ 5mm（够 WS2812 模块的灯珠透光）
  - 后盖可拆，方便塞电路板和理线
  - 留 USB 线出口（一般底部或背后开个 8×8mm 的方孔）
- 如果你有朋友有 3D 打印机，去 [Thingiverse](https://www.thingiverse.com) 搜 `traffic light` 免费下模型，找个有空腔的版本

### 6. 可选小物

| 物品 | 用途 | 价格 | 建议 |
|---|---|---|---|
| 面包板（小号 170 孔） | 调试期临时插线 | ¥3 | **推荐**，调通前不要焊死 |
| 470Ω 电阻 | WS2812 DIN 保护 | ¥1 | 3 颗灯量级可省 |

## 二、接线图

### 引脚对照

```
                    Raspberry Pi Pico (顶视图，USB 在上)
                    ┌──────[Micro USB]──────┐
                GP0 │ 1                  40 │ VBUS  (5V, 来自 USB)
                GP1 │ 2                  39 │ VSYS
                GND │ 3                  38 │ GND
                ... │                    ... │
                    │                       │
                    │       (中间省略)       │
                    │                       │
                GP15│ 20                 21 │ 3V3(OUT)  ← 给 WS2812 供电
                    └───────────────────────┘
                         ↑ 用这两个 + GND
```

**实际只用 3 个引脚**：
- **3V3(OUT)**（物理脚 36）→ WS2812 的 **VCC**（3 颗都接，并联）
- **GND**（物理脚 38，或任意 GND）→ WS2812 的 **GND**（3 颗都接，并联）
- **GP15**（物理脚 20）→ 第一颗 WS2812 的 **DIN**

> 💡 WS2812 标称 5V 供电，但 3 颗灯用 3.3V 也能正常亮（亮度略低，反而护眼）。用 3.3V 的好处是 Pico GPIO 输出也是 3.3V，**电平直接匹配，省一个电平转换芯片**。如果发现亮度不够再考虑换 5V（VBUS 引脚，物理脚 40）。

### 三颗灯串联接法

   ┌─────┐              ┌─────┐             ┌─────┐             ┌─────┐
   │ 3V3 ├──┬──────────┤ VCC │       ┌────┤ VCC │       ┌────┤ VCC │
   │     │  │           │     │       │     │     │       │     │     │
   │ GND ├──┼──┬───────┤ GND │       │ ┌──┤ GND │       │ ┌──┤ GND │
   │     │  │  │        │     │       │ │   │     │       │ │   │     │

   │     │  │  │        │     │       │ │   │     │       │ │   │     │

### 接线（清晰版）

**电源（VCC 和 GND 是"公共线"，三颗灯都要接到 Pico 的同一个 3V3 和 GND）：**
- Pico `3V3` ─┬─ LED1.VCC
- &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;├─ LED2.VCC
- &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└─ LED3.VCC
- Pico `GND` ─┬─ LED1.GND
- &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;├─ LED2.GND
- &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└─ LED3.GND

**数据（菊花链，只接一根线，依次往下传）：**
- Pico `GP15` → LED1.DIN
- LED1.DOUT → LED2.DIN
- LED2.DOUT → LED3.DIN
- LED3.DOUT → 悬空不接

> 用面包板会非常方便：把 3V3 和 GND 各引一根线插到面包板的电源轨上，三颗灯的 VCC/GND 都插到这两条公共轨即可。

### 怎么把 3 根公共线塞进 Pico 一个引脚？

两个办法：
- **A. 面包板**：Pico 的 3V3 引脚先连到面包板电源轨，3 颗 LED 的 VCC 都插到电源轨。GND 同理。
- **B. 直接缠绕**：把 3 颗灯的 VCC 杜邦线公头先拧在一起再插进 Pico。临时调试可以，作品最终版还是用面包板或小型 PCB 板。

## 三、到货后的验收顺序

按这个顺序做，每步出问题都好定位：

1. **裸 Pico 上电测试**：USB 插电脑，看 Pico 板上的绿色板载 LED 是否常亮 → 证明板子和线没坏
2. **烧录 MicroPython 固件**（5 分钟，详细步骤代码到位后再讲）
3. **只接 1 颗 WS2812**，跑"全红→全绿→全蓝"测试 → 证明灯和接线 OK
4. **接齐 3 颗**，跑"依次点亮"测试 → 证明菊花链 OK
5. **接 Claude Code hook**，跑一次对话看灯变化 → 收工
